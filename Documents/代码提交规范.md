# 代码提交规范

## 提交格式

模块名: 一句话描述改动  
特殊类型（BUMP / BUILD / BUGFIX）可放在模块名前或独立使用

示例：

APP: 新增 SQL 编辑器自动补全功能  
DRIVER: 优化连接池初始化逻辑  
APP/DRIVER: 调整连接创建流程  
BUMP: 更新项目版本至 v1.0.0-arch.1  
BUILD: 优化构建脚本

## 模块命名规则

1. 模块名必须使用 **大写英文**。
2. 模块名使用 **单数形式**，不要使用复数。  
   例如模块目录为 `drivers`，提交时写 `DRIVER`。
3. 如果一次提交涉及多个模块，**最多填写两个影响最大的模块**。

示例：

APP/DRIVER: 重构连接创建流程

## Bug 修复

Bug 修复必须使用 `BUGFIX` 前缀。

示例：

BUGFIX: 修复 SQL 编辑器 Command+S 保存跳转问题  
BUGFIX: 修复连接关闭后重新打开提示数据源已关闭

## 版本号更新

版本号变更必须使用 `BUMP` 前缀。

示例：

BUMP: 更新项目版本至 v1.0.0-arch.1

## 构建相关

构建脚本、CI/CD 配置、依赖更新等使用 `BUILD` 前缀。

示例：

BUILD: 优化 Maven 构建脚本  
BUILD: 更新依赖版本

## 编写要求

1. 描述使用 **一句完整语义的短句**。
2. 描述应 **直接说明改动内容或修复的问题**。
3. 避免模糊描述，例如：

**不推荐：**

APP: 修改代码  
APP: 一些优化

**推荐：**

APP: 新增查询结果分页功能  
APP: 优化表结构加载性能  
BUGFIX: 修复无主键表删除数据异常  
BUMP: 更新项目版本至 v1.0.0-arch.1  
BUILD: 优化构建脚本
